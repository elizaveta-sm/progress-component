# Прототип блока Progress

Основное предназначение - отображать процесс выполнения процессов и их прогресс выполнения.

Демо: https://elizaveta-sm.github.io/progress-component/

## Архитектура

- **Progress** - отвечает за отображение и управление состояниями
- **Controls** - отвечают за пользовательский ввод
- **main.js** - связывает UI и логику

## API блока Progress

Блок Progress создается через функцию:

`const progress = createProgress(rootElement);`

**Методы**

- `setValue(value: number)` устанавливает значение прогресса от 0 до 100.
- `setAnimated(isAnimated: boolean)` включает и выключает анимацию.

  Во время анимации:
  - прогресс вращается по часовой стрелке
  - ввод значения блокируется
  - компонент работает как loader

- `setHidden(isHidden: boolean)` скрывает или отображает блок.

  Когда он скрыт со страницы - верстка не рушится.

## Реализация прогресса

Дуга прогресса представляет собой SVG круг.

Для крайних значений (0 и 100) прописаны специальные условия, так как иначе рендеринг SVG при 0 показывает небольшую часть дуги, а при 100 - отрезает.

SVG создается динамически внутри компонента через шаблон (template), поэтому компонент можно переиспользовать без копирования HTML разметки.

## UI

**В Controls реализованы:**

- Ввод значения прогресса с проверкой на то, что ввод - число.

  При попытке ввести значение меньше 0, оно нормализуется до 0, а значение больше 100 - до 100.

- Тумблер (toggle) анимационного режима.
- Тумблер (toggle) видимости блока.

UI-контролы не содержат логики и работают через API блока Progress.

**Шрифт:** Plus Jakarta Sans. Был выбран по наибольшей схожести со шрифтом из макета.

## Адаптивность

Блок адаптируется под ориентацию экрана: на Portrait элементы располагаются вертикально, на Landscape - горизонтально.
